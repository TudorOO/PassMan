PASSMAN

Flask-based password manager


Passman password manager provides a safe way to create and store incredibly strong passwords for all of your apps. It provides safety and convenience, removing the need to remember an ever-growing number of passwords, while also making sure that they are complex enough to be safe in the event of a breach.



Why it's safe:


1. The master password is never stored on the server, not even temporarily.

This makes sure that, even if our server is breached, your passwords will still be safe.


2. The passwords are stored using the latest encryption technology available.

We hash the master password with Argon2id and encrypt the passwords using a key derived from the master password with AESGCM. These technologies provide powerful protection against even the strongest attacks, such as GPU-Based ones.


3. Encryption and decryption of passwords can happen either only server-side, or both client-side and server-side.

You get to choose! Do you trust your machine more or our server more? Whichever the case, you can have it your way: minimal-knowledge or zero-knowledge!


4. No AI corners cut.

All code in this project is written and audited by hand, without AI-generated components, to ensure full transparency and control over security logic.


5. No-password login.

If you forget your master password, no worries! You can download a backup file which you can then use to login without the password.

WARNING: While the backup file is encrypted, we do not provide any protection in the case that a nefarious actor gets a hold of it. You should store it incredibly safely, ideally in an USB or another location not connected to the internet.



How it works:


1. You make an account

The master password you provide is encrypted using scrypt and stored. In addition, a backup file is generated, which you can then download and use if you wish to login without a password later.


2. You create a passkey for an app

The passkey you create is encrypted with AES-GCM with a key derived from the master password using Argon2id which is not ever stored on the disk. This passkey can then be revealed at any time by inputting the master password again. All encryption is done with random nonces and all derivation is done with user-specific salts.



Other features(some not yet added):

1. Resetting passwords
2. Two Factor Authentication
3. Zero-knowledge mode(client+server side AES-GCM encryption/decryption)
4. Breach monitoring
5. YubiKey integration


Tech Stack: Flask, HTMX, SQLite, Bootstrap, Argon2id, AES-GCM, Flask-WTF CSRF Protection, Flask-Limiter, Flask-talisman





Disclaimer: Passman is a learning project and should not be used for real-world password storage.
