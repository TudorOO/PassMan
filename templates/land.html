PASSMAN

Flask-based password manager


Passman password manager provides a safe way to create and store incredibly strong passwords for all of your apps. It provides safety and convenience, removing the need to remember an ever-growing number of passwords, while also making sure that they are complex enough to be safe in the event of a breach, using a full zero-knowledge arhitecture with both CSE and SSE.



Why it's safe:


1. The server doesen't see your password, ever.

From the moment you enter it, the password is encrypted before being sent to our server using the latest and strongest technologies available. This protects against man-in-the middle attacks and ensures that even if the server is compromised, your master password is safe.


2. All data is stored using the latest encryption technology available.

We hash the master password server-side with scrypt and encrypt the passwords using a key derived from the master password(with Argon2id) and AESGCM. Your email and username are also stored encrypted, so a breach won't give away any personal info. These technologies provide powerful protection against even the strongest attacks, such as GPU-Based ones.


3. Encryption and decryption of passwords can happen either only server-side, or both client-side and server-side.

You get to choose! By default, your data is encrypted both client-side and server-side. This provides maximum security, as the server doese't see your data even once(zero-knowledge). The other option, server-only encryption, makes it so that the server processes your password in plaintext for a miniscule amount of time, however it makes it way easier to setup on older versions of browsers and the tradeoff in security is not that great. Wheather you want complete security or ease of use, you can have it your way!


4. No AI corners cut.

All code in this project is written and audited by hand, without AI-generated components, to ensure full transparency and control over security logic.


5. No-password login.

If you forget your master password, no worries! You can download a backup file which you can then use to login without the password.

WARNING: While the backup file is encrypted, we do not provide any protection in the case that a nefarious actor gets a hold of it. You should store it incredibly safely, ideally in an USB or another location not connected to the internet.



How it works:


1. You make an account

The master password you provide is encrypted server-side using a key derived from itself via Argon2id with AES-GCM and then encrypted server-side using scrypt and stored. In addition, a backup file is generated, which you can then download and use if you wish to login without a password later.


2. You create a passkey for an app

The passkey you create is encrypted with AES-GCM with a key derived from the master password using Argon2id which is not ever stored on the disk. This passkey can then be revealed at any time by inputting the master password again. All encryption is done with random nonces and all derivation is done with user-specific salts.



Other features(some not yet added):

1. Resetting passwords
2. Two Factor Authentication
4. Breach monitoring
5. YubiKey integration


Tech Stack: Flask, HTMX, SubtleCrypto, SQLite, Bootstrap, Argon2id, AES-GCM, Flask-WTF CSRF Protection, Flask-Limiter, Flask-talisman, ZXCVBN





Disclaimer: Passman is a learning project and should not be used for real-world password storage.
